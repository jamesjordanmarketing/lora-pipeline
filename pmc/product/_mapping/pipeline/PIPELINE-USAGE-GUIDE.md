# Spec Extension & Segmentation Pipeline - Usage Guide

**Version:** 2.0  
**Date:** December 24, 2025  
**Purpose:** Complete guide for running the extension pipeline to transform structured specs into execution-ready prompts

---

## OVERVIEW

This pipeline transforms a generic structured specification into progressive execution prompts by analyzing the existing codebase and generating extension documentation.

**Three Stages:**
1. **Stage 0 (GENERATE PROMPT)**: Create custom integration prompt from generic template
2. **Stage 1 (ANALYZE)**: Execute prompt with AI to generate extension docs (Infrastructure Inventory, Extension Strategy, Implementation Guide)
3. **Stage 2 (SEGMENT)**: Break down features â†’ progressive execution prompts

**Version:** 2.0 (Updated workflow using meta-prompt approach)

---

## PREREQUISITES

### Required Files (Inputs)

Before running the pipeline, ensure you have:

1. **Generic Meta-Prompt Template**
   - Path: `pmc/product/_prompt_engineering/04d-integrate-existing-codebase_v2.md`
   - Contains: Product-agnostic template for extension analysis

2. **Structured Specification** (04c)
   - Path: `pmc/product/_mapping/pipeline/04c-pipeline-structured-from-wireframe_v1.md`
   - Contains: 7 sections of features with generic infrastructure

3. **Existing Codebase**
   - Path: `src/` (or your project's source directory)
   - Contains: Production code with existing infrastructure

### Files Generated by Pipeline (Outputs)

The pipeline will generate:

1. **Custom Integration Prompt** (04e)
   - Generated in Stage 0
   - Used to execute AI analysis in Stage 1

2. **Infrastructure Inventory** (04d)
   - Generated by AI in Stage 1
   - Documents what exists in codebase to USE

3. **Extension Strategy** (04d)
   - Generated by AI in Stage 1
   - Documents how features map to existing infrastructure

4. **Implementation Guide** (04d)
   - Generated by AI in Stage 1
   - Provides exact code patterns to follow

5. **Execution Prompts** (04f)
   - Generated in Stage 3
   - Progressive prompts for implementation

### Required Tools

- Node.js (v14 or higher)
- Access to the repository files
- AI assistant (Claude, ChatGPT, etc.) for Stage 1 execution

## STAGE 1: EXECUTE INTEGRATION MERGE (Manual AI Execution)

### Purpose
Use the generated prompt to merge the three 04d documents into a single integrated extension specification.

### Input
- Integration prompt: `04e-[product-abbrev]-merge-integration-prompt_v1.md`
- Infrastructure Inventory (04d)
- Extension Strategy (04d)
- Implementation Guide (04d)

### Output File
- `04e-[product-abbrev]-integrated-extension-spec_v1.md` - Unified specification (~3,000-5,000 lines)

### Process

1. **Open the integration prompt**: Read `04e-[product-abbrev]-merge-integration-prompt_v1.md`
2. **Execute with AI**: Provide the prompt to an AI assistant (Claude, ChatGPT, etc.)
3. **AI merges documents**: AI combines all three 04d documents into one comprehensive spec
4. **Save output**: Save AI's response to the specified output path

### Validation

After AI execution, verify:

1. **Output file exists**:
   - `04e-[product-abbrev]-integrated-extension-spec_v1.md`

2. **File size is reasonable**:
   - Integrated Spec: ~100-200 KB (3,000-5,000 lines)

3. **Content quality**:
   - All infrastructure substitutions applied
   - Extension framing maintained
   - Implementation patterns are specific and exact
   - Ready for segmentation

## TROUBLESHOOTING

### Stage 1 Issues

**Problem**: "Error reading file"
- **Cause**: File path incorrect or file doesn't exist
- **Solution**: Verify all input file paths are correct
```bash
ls -lh "../_mapping/pipeline/04c-pipeline-structured-from-wireframe_v1.md"
ls -lh "../_mapping/pipeline/_run-prompts/04d-infrastructure-inventory_v1.md"
ls -lh "../_mapping/pipeline/_run-prompts/04d-extension-strategy_v1.md"
ls -lh "../_mapping/pipeline/_run-prompts/04d-implementation-guide_v1.md"
```

**Problem**: "Found 0 sections"
- **Cause**: Structured spec format doesn't match expected pattern
- **Solution**: Check that spec has sections formatted as `## SECTION N: Title`

**Problem**: Output file is very small
- **Cause**: Transformation failed silently
- **Solution**: Check console output for errors, verify input files are correct format

### Stage 2 Issues

**Problem**: "Error reading file" for integrated spec
- **Cause**: Stage 1 didn't complete successfully
- **Solution**: Re-run Stage 1 and verify output file exists

**Problem**: "Found 0 sections"
- **Cause**: Integrated spec format doesn't match expected pattern
- **Solution**: Verify Stage 1 output, check for `## SECTION N: Title - INTEGRATED`

**Problem**: No prompts generated
- **Cause**: No features extracted from sections
- **Solution**: Check integrated spec has features formatted as `#### FR-X.Y: Title`

**Problem**: Too few prompts
- **Cause**: Features not categorized correctly
- **Solution**: Review feature categorization logic in script

### General Issues

**Problem**: Permission denied
- **Solution**: Check directory permissions, run with appropriate privileges

**Problem**: Node.js not found
- **Solution**: Install Node.js v14+ or add to PATH

**Problem**: Unexpected output
- **Solution**: Check all input files are the correct version (v1)
